# MVC vs MVT

## MVC - Model View Controller [Spring, Nodejs, PHP, Ruby, etc]

### Model
1. Model is a class.
2. A Model is the standard of specific type of data that your backend will accept for processing.
3. A model is direcly related to Database (in most cases, but not mandatory).

### View [Not relevant]
1. A view is a file which contains UI elements.
2. A view is returned as a response of a API.
3. It is usally a HTML or an HTML looking like file, eg: JSP, ASPX, EJS, Jinja, etc.

### Controller [Very Important]
1. A controller is a file, whose whole job is to accept requests and return responses.
2. A controller's responsibility is to trigger the processing of the request and send the correct response.
